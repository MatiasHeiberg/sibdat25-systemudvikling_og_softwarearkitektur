03_07 — Med Command

Undo/redo via ICommand og CalculatorInvoker

Beskrivelse

Command Pattern indkapsler hver operation som et objekt. Hvert command kender sin Execute() og Undo(). CalculatorInvoker holder en undo-stack og redo-stack.

Klassediagram — Systemet
Hold musen over en klasse for at se pseudo kode
Grøn Byg Command-systemet

Projektnavn: Exercise_03_07_MedCommand

Domain — ICommand

Et interface ICommand med to metoder:

  • void Execute() — udfører operationen
  • void Undo() — fortryder operationen

Domain — Calculator

Klassen Calculator har kun én property: CurrentValue af typen double med public get og public set. Starter på 0. Ingen metoder — al logik ligger i command-klasserne.

Domain — Command-klasser

Fire klasser der alle implementerer ICommand. Hver command har to private readonly fields: _calculator (Calculator) og _value (double), som sættes via constructor.

  • AddCommand — Execute() lægger _value til CurrentValue, Undo() trækker den fra
  • SubtractCommand — Execute() trækker _value fra, Undo() lægger den til
  • MultiplyCommand — Execute() ganger med _value, Undo() dividerer
  • DivideCommand — Execute() dividerer med _value, Undo() ganger. Constructor kaster DivideByZeroException hvis value er 0

Application — CalculatorInvoker

Klassen CalculatorInvoker styrer undo/redo med to stacks:

  • _undoStack — Stack<ICommand>
  • _redoStack — Stack<ICommand>

Metoder:

  • ExecuteCommand(ICommand command) — kalder command.Execute(), pusher til undo-stack, og rydder redo-stack
  • Undo() — returnerer bool. Popper fra undo-stack, kalder Undo(), pusher til redo-stack. Returnerer false hvis undo-stack er tom
  • Redo() — returnerer bool. Popper fra redo-stack, kalder Execute(), pusher til undo-stack. Returnerer false hvis redo-stack er tom

Properties: CanUndo og CanRedo (bool, readonly) der tjekker om de respektive stacks har elementer.

UI — ConsoleUI

ConsoleUI modtager både Calculator og CalculatorInvoker via constructor injection.

Metoden Run() kører et loop der læser input:

  • "+5" → opretter new AddCommand(calculator, 5) og kalder invoker.ExecuteCommand(cmd)
  • "-3", "*2", "/4" → tilsvarende for Subtract, Multiply, Divide
  • "undo" → kalder invoker.Undo()
  • "redo" → kalder invoker.Redo()
  • "reset" → sætter CurrentValue = 0 direkte
  • "quit" → afslutter programmet

Program.cs

Composition root:

Program.cs
var calculator = new Calculator();
var invoker = new CalculatorInvoker();
var ui = new ConsoleUI(calculator, invoker);
ui.Run();

Rød MacroCommand — optag og afspil

Udvid systemet med en MacroCommand der kan optage en sekvens af kommandoer og afspille dem alle på én gang. Når brugeren fortryder en macro, skal alle operationer i sekvensen fortrydes i omvendt rækkefølge.

Domain — MacroCommand

Klassen MacroCommand implementerer ICommand og holder en List<ICommand> med de optagne kommandoer.

  • Execute() — kalder Execute() på alle kommandoer i listen (i rækkefølge)
  • Undo() — kalder Undo() på alle kommandoer i omvendt rækkefølge

ConsoleUI — nye kommandoer

  • "record" → starter optagelse. Alle følgende operationer (+5, -3 osv.) tilføjes til en liste i stedet for at blive udført
  • "stop" → stopper optagelse og gemmer listen som en MacroCommand
  • "play" → udfører den gemte MacroCommand via invoker.ExecuteCommand(macro)

Fordi MacroCommand implementerer ICommand, virker undo/redo automatisk — én Undo() fortryder hele makroen.

Eksempel

Konsol
[0] > record
  Optagelse startet...
[0] > +10
  Optaget: +10
[0] > *3
  Optaget: *3
[0] > -5
  Optaget: -5
[0] > stop
  Macro gemt (3 kommandoer)
[0] > play
  = 25
[25] > undo
  Undo => 0
