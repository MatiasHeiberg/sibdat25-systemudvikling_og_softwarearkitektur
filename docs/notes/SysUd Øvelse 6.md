# Verificering 
	Kundeønske:
	- Fitnesscenteret ønsker at kunne lancere en første version inden for 3-4 måneder for at følge med konkurrenterne.
	
	Dialog i grupperne om:
	- Hvad er Core User Flow i systemet?
		- Hvad er den centrale aktivitet brugeren gentager igen og igen i systemet?
	- Hvilke features er der når systemet udgives første gang?
		- Hvilket produkt kan vi lave som overholder kravene, men med mindst mulig arbejde? (Minimum Viable Product)
	- Hvordan prioritere I arbejdet?
		- Hvilke række følger bliver de lavet I, og hvorfor (Prioritering)?
	Afklaring (spørgsmål I kan anvende internt):
	- Er vi enige om hvad systemet skal kunne?
	- Er vi enige om prioriteringen?
	- Har vi samme forståelse af mål og scope?
	- Har nogen ændret mening under diskussionen?
